Git のコミットメッセージの種別

Git を使っていて、日本語のコミットメッセージの先頭に [add] や [fix] などの種別をつけてわかりやすくしているのをよく見かけます。

そのルールに合わせるときに、毎回種別のそれぞれの意味を調べてしまうのでここにまとめます。

コミットメッセージの種別

  • fix：バグの修正
  • add：新規機能やファイルの追加
  • update：機能の修正
  • change：仕様の変更
  • clean：整理
  • disable：無効化
  • remove：ファイルなどの削除
  • upgrade：バージョンアップ
  • revert：変更の取消
